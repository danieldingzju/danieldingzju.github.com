hacking fastjson

背景

fastjson是一个很牛逼的开源json工具包，谁用谁知道。它有如下常见用法：

  1. 假设我们的json数据结构为：

{
id: xxx
name : xxx
address : xxx
phone : xxx
}

  1. 定义json数据结构对应的POJO对象，假设为(请允许我用lombok偷个懒)：

@Data
Class PeopleDO {
private Long id;
private String name;
private String phone;
}

  1. 然后我们使用API把json数据解析为对应的POJO实例：

com.alibaba.fastjson.JSON.parseObject(json_data, PeopleDO.class)

---

问题

在我的hercules 的agent 开发过程中，我发现了cpu，load飙高，用jstack一看，得到了

![图片](http://tempwork.qiniudn.com/fastjson_current_problem.png)

关于如何定位问题，请看[classLoader 性能问题](http://www.atatech.org/articles/49976?msgid=726987)

---

解决

首先，有一个餐前点心知识点是：JDK6的ClassLoader使用大量的syncronized关键字。

让我们一起看下代码，从**parseObject**方法跟进去，并且找到使用到ASMClassLoader.defineClassPublic()的类

这里其实可以明白的看到 code1 这个变量其实每次调用parseObject进来之后都是会通过asm解析，改造生成同样内容的byte[]的；同时这个函数的目的是为了拿到 exampleClass 的实例；那么我们可以通过缓存 exampleClass 对象来避免反复调用 this.classLoader.defineClassPublic 这个方法，最终解决并发问题。
